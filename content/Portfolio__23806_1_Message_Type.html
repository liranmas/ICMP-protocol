Message Type

ICMP Message Types are divided into two categories, Error-reporting and Query.

Error-reporting

Serves as an error reporter for the router or destination host when delivering one specific IP packet.
ICMP reports errors, but no correction has occurred, and the error messages are sent back to the source where the packet is sent.

Query

ICMP may be used to request and fetch information about a packet from the router or the host, which is for diagnostics or debugging purposes for the connection or the route itself.

Example of Error-reporting messages: (Type 3) Destination Network Unreachable

The ICMP Type 3 error-reporting message "Destination Unreachable" is sent to the source host when it encounters an error reaching the destination host. In total, there are seven different codes (0, 1, 2, 3, 7, 9, 10) where each code type represents different types of delivery problems.

Example of Query messages: (Type 8, 0) Echo Request and Reply

The ICMP Echo Request Type 8 and Echo-Reply Type 0 is used for diagnostic purposes when trying to test the connectivity between two remote hosts. The messages will execute the following upon successful echo replies to the echo requests, which indicates a successful test between the hosts. It assumes that a communication path exists between the hosts and assumes that the end host function through the network layer.
