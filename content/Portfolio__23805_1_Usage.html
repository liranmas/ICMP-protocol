Usage

Error Message

When different communication devices connect through the internet, there are situations where the data cannot reach its destination IP address. ICMP would generate an error message back to the source of the data. An example would be when a packet is too large for a router, and the packet would be dropped by the router and send an ICMP message back to the sender.

Network Diagnostics

Traceroute utility was used with ICMP protocol to display the routing path of connected devices through the internet. This path is the physical path the data takes to arrive at the destination IP. This utility helps IT professionals to understand and debug any source of error in network performance.

Network performance testing

Using Ping utility to test the connection speed between communication devices by reporting how long it takes for a packet to reach the destination and return to the sender. This tool helps understand the latency between devices and is also commonly used by non-IT professionals to check for their local devices' latency with network providers.

Message Syntax


The ICMP Message Syntax consists of an 8-byte header along with the variable size data section. The first 8 bits in the header indicates the type of message. The next 8 bits in the header is a code field which provides detailed reasoning to a specific message type. (i.e., Type 3, Code 0 represents Destination Network Unreachable. Type3, Code 1 represents Destination Host Unreachable). The last 16 bits in the header are the sum of the entire ICMP message. As explained above in Message Type, the diagram's data section can be either one of the error messages or query messages.
